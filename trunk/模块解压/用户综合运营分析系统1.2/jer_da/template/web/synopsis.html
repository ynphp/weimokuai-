趋势

概述

趋势做为整数据分析模块的用户增长统计表。尽可能的帮助运营者理清系统中的粉丝关注数据，用更直观的图表方式来展现数据。

运营者可以根据预设时间段 7天、14天、30天、60天及自定义时间段来查看不同时间段的粉丝增长信息。

事件

概述

事件可以简单理解为某一时间段内发生的事情或者举办的活动，并对该事件的粉丝增长影响和特性做分析统计。

使用
  1. 要使用事件分析则先 添加事件分类。事件分类只用做区分不同类型的事件，以便于管理。
  2. 事件分类添加完毕后，点击 新建事件，并按照表单依次填写。
  3. 新建完毕后，在 事件列表 中点击 分析，进入到事件分析页面。
概述

模块分析主要针对第三方模块数据表及自定义的数据表（多个系统的数据表）进行统计分析。以自定模块的数据表作为基础数据，对比某一时间段内的粉丝增长趋势，并做出分析和决策。

使用
  1. 要使用模块分析则先 添加模块。模块只用做区分不同类型的数据表，以便于管理。
  2. 模块添加完毕后，点击 表管理。进入到 表管理 后再点击 添加模块表，并按照表单依次填写。
  3. 在 添加模块表 里，对应数据表 为需要进行分析的数据表；对应数据表字段 为需要分析的字段；数据表时间字段 为数据创建的时间戳。
  4. 新建完毕后，在 模块管理 中点击 分析，进入到模块分析页面。
（如无特殊需求，添加模块表中的对应数据表字段，请用表的主键）
